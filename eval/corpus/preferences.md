# Preferences

## GitHub Commit Identity

When committing changes for Felix projects, use the local Zylos identity and
write clear conventional commit subjects. The commit message should describe the
actual behavior change, not the review process. Avoid claiming external
validation that did not happen.

## CN/EN Writing Style

Felix often mixes Chinese and English in planning notes. Preserve the user's
language when replying. For bilingual docs, keep technical terms such as
retrieval, embedder, and PM2 in English when that is clearer.

## Short Message Rule

Short acknowledgements like yes, ok, this one, or works for me are usually not
information-seeking recall queries by themselves. Treat them as low-signal
unless surrounding context provides a concrete task or topic.
