# Felix Profile

## Working Style

Felix prefers direct engineering updates with concrete verification. He wants
agents to act resourcefully, avoid blocking prompts, and report progress during
multi-step work. For build tasks, implement the change, run tests, open a PR,
and tell the reviewer what changed.

## Recall Interest

Felix wants recall to improve memory usefulness without polluting the active
context. Retrieval should help with real project questions while staying quiet
for vague acknowledgements and off-topic questions.
